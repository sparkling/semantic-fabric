## TL;DR

`semantic-fabric` is a **Rust virtualisation-only OBDA engine**: it answers
**SPARQL 1.2 by rewriting to SQL** over live relational databases executing
**R2RML** mappings ‚Äî **no JVM, no materialisation** (instance data is generated on
demand, streamed, discarded; only the ontology `T` + mapping graph `M` live in
memory). It is **built and working**: the rewriter, both executors (SQLite +
PostgreSQL), a live SPARQL 1.2 Protocol HTTP endpoint, property paths, W3C
conformance, and a benchmark/comparison harness all run end to end.

## What is built

- **Rewriter (`sf-sparql`)** ‚Äî parse (spargebra) ‚Üí T-saturation ‚Üí unfold/ISWC-2018
  base translation ‚Üí 6-rule cascade ‚Üí dialect SQL ‚Üí stream (ADR-0003/0007/0008).
- **Two executors** ‚Äî `exec.rs` (sync SQLite) and `exec_pg.rs` (async PostgreSQL,
  `select_pg`/`ask_pg`/`construct`/`dump` over a bounded-memory `PgRowStream`).
- **`serve` HTTP endpoint (`sf-serve`)** ‚Äî SPARQL 1.2 Protocol (`GET`/`POST
  /sparql`); content-negotiated (SPARQL Results JSON/XML/CSV/TSV;
  Turtle/N-Triples/JSON-LD); **streamed row-by-row** from the cursor into the HTTP
  body; governed (timeout, max-query-len, error‚Üístatus). `sf-cli serve --source
  sqlite:<path>|pg:<conninfo> --mapping <ttl> [--ontology <ttl>] [--bind addr]`
  (ADR-0019 G8).
- **Property paths** ‚Äî `^p`, `p/q`, `p|q`, `p?`, `!p`, composite `+`/`*`, gated by
  the spareval `=_bag` differential (ADR-0007/0008).
- **Named-graph quad output**, **R2RML ¬ß10 datatype canonicalisation**,
  **constant-memory streaming** (peak engine heap byte-constant across scale).
- **Conformance harness (`sf-conformance`)** ‚Äî vendored W3C RDB2RDF via CONSTRUCT,
  blank-node graph-iso, a real spareval differential oracle + NoREC, rudof-Native
  M‚ãàT SHACL gate, EARL reports (ADR-0005).
- **Benchmark + comparison** ‚Äî `sf-bench` (GTFS-Madrid OBDA + the constant-memory
  demo); [`BENCHMARKS.md`](../BENCHMARKS.md); [`COMPARISON.md`](../COMPARISON.md)
  (honest, reproducible head-to-head vs Ontop + Morph-KGC).

## Outstanding / not done (honest)

- **`R2RMLTC0002f`** ‚Äî documented deviation, **won't-fix by design**: per R2RML ¬ß5
  (SQL:2008) strict identifier folding would be unsound for a virtualiser (intro¬≠
  spection erases delimited-vs-regular provenance) **and** net-negative on
  conformance (it would fail the suite's own positive cases). Tracked as a
  gate-excluded expected deviation (`sf_conformance::EXPECTED_DEVIATIONS`, ADR-0015).
- **Property-path 501 residuals** ‚Äî bound endpoints, nested closures inside a
  composite, shape-mismatched composites, and `p?`/`P*` reflexive over
  multi-predicate/composite graphs return a documented `501` (never silently wrong).
- **`Q5`/`OPTIONAL` latency at scale** ‚Äî a *real* perf gap: sf's left-join plan
  scales ~linearly worse than Ontop's at 10√ó (see COMPARISON.md ¬ß1). The clearest
  next optimisation target (the cascade's OPTIONAL handling).
- **ADR-0014 production-hardening backlog** ‚Äî reliability/retries/circuit-breakers,
  TLS/secrets/rate-limiting, packaging (container/k8s/probes), horizontal scale,
  lifecycle (mapping/ontology hot-reload + schema-drift), observability wiring.
  Relevant now that `serve` exists.
- **Out of scope by charter (ADR-0002)** ‚Äî heterogeneous CSV/JSON/XML sources,
  external `SERVICE` federation, FNML. (A different product; revisit only on a pivot.)
- **Measure/need-driven deferrals (ADR-0020)** ‚Äî cost-driven OBDA/JUCQ, SIMD
  term-gen, SQLite `INSTR`-based case-sensitive FTS, D016 `bytea` PG forked fixtures
  (6 PG skips). FTS baseline is PostgreSQL `LIKE` only today.
- **Tidy:** `crates/sf-sparql/src/emit.rs` is ~612 lines (>500 guideline) ‚Äî a
  cosmetic split is deferred.

## How to resume

```bash
cargo build --workspace
cargo test  --workspace                     # 181 passing
cargo fmt --all --check                     # CI gate
cargo clippy --workspace --all-targets -- -D warnings

cargo run -p sf-cli -- conformance          # W3C RDB2RDF (SQLite) ‚Üí 81/82
SF_PG_URL='host=localhost port=5432 user=<you>' \
  cargo test -p sf-conformance --test w3c_pg_suite -- --nocapture   # PG ‚Üí 80/81

# serve a live SPARQL endpoint (SQLite or PostgreSQL)
cargo run -p sf-cli -- serve --source sqlite:data.db --mapping mapping.r2rml.ttl --bind 127.0.0.1:7878
curl --data-urlencode 'query=SELECT * WHERE {?s ?p ?o} LIMIT 5' \
  -H 'Accept: application/sparql-results+json' http://127.0.0.1:7878/sparql

cargo bench -p sf-bench                      # GTFS-Madrid OBDA + constant-memory
# fair head-to-head vs Ontop (see COMPARISON.md "Reproduce everything"):
ONTOP_HOME=$PWD/ontop-cli scripts/compare/race.sh 1 25
```

## Working discipline (how this was built, for continuity)

Each capability landed as a gated increment: implement ‚Üí verify (build/fmt/clippy/
tests + conformance non-regression) ‚Üí **adversarial review** ‚Üí conditional fix. The
review caught real defects every wave (a re-tripped fmt gate, a serve SELECT that
buffered the whole result, an NPS bag-semantics undercount, an OPTIONAL `=_bag`
bug) ‚Äî keep that loop. Correctness is gated by the spareval differential + NoREC +
the W3C conformance floor; performance work must hold those gates (no speed for
correctness). No fabricated numbers in any doc ‚Äî real run (shown command) or cited.
