## TL;DR

`semantic-fabric` is a greenfield **Rust virtualisation-only OBDA engine**: it answers **SPARQL 1.2 by rewriting to SQL** over **live relational** databases executing **R2RML** mappings, replacing the host platform's **Jena (Fuseki + jena-shacl)** with **no JVM** on the runtime path. **Instance data is never materialised** — generated on demand, streamed, discarded. The design is **complete and committed**, RDF 1.2 / SPARQL 1.2 readiness is validated, and the four foundational cost/term-gen optimisations are now baked into the design — **but all engine logic is still `todo!()`**. Next: **build the virtualiser** — enable the 1.2 feature flags + wire deps, then the SPARQL 1.2→SQL rewriter core.

## Load-bearing decisions (the spine)

1. **Virtualisation-only** — instance data never materialised; OBDA ⟨T,M⟩ (ontology + mapping graph) held in memory (ADR-0002/0003/0004).
2. **Streaming bounded-memory invariant** — the source DB does the set-work + spills natively; engine memory bounded by ⟨T,M⟩ + a fixed budget, independent of data; no OLAP/columnar on the relational path; cross-source = bounded semi-join + k-way merge (ADR-0006/0010).
3. **RDF 1.2 / SPARQL 1.2, no JVM** — Oxigraph crates (parser + terms, **not** the store/evaluator) + rudof SHACL Native; replaces Jena end-to-end (ADR-0001/0004/0019).
4. **SPARQL→SQL = ISWC-2018 base translation** (unoptimised ground truth) + order-disciplined cascade, verified by NoREC internal-differential + MR1 metamorphic + in-memory oracle (ADR-0007/0012).
5. **Entailment in the rewriter** — tier-1 hierarchy folding (subclass/subproperty/inverse/symmetric; transitive via `P+`/`P*`); tier-2 OWL-QL **moot — depth-0 by construction** (the OWL-as-documentation policy excludes `owl:Restriction`/existentials) (ADR-0008).

## 1.2 readiness (ADR-0019) — the #1 build prerequisite

Oxigraph passes the full W3C RDF 1.2 + SPARQL 1.2 test suites (empty skip-lists); rudof has SHACL 1.0 Core (Native) + `sh:sparql` — **but all 1.2 is compiled OUT today**. First build step: enable `rdf-12`+`rdfc-10` (oxrdf), `sparql-12`+`sep-0002`+**`sep-0006`**+`standard-unicode-escaping` (spargebra), wire oxttl/sparesults, **add `oxjsonld` ≥0.2.5**. **`LATERAL` (`sep-0006`)** is enabled as a documented opt-in extension, kept out of the 1.2-conformance surface (ADR-0007/0019). Upstream-contribution candidates (not blockers; Native mode avoids): rudof G4 (property-pair SPARQL-mode) + G6 (two user-reachable panics); G2/G5 already shipped upstream; G7 (SHACL 1.2) = wait.

## What to implement now (SOTA items — ADR-0020)

Most items are **gated/deferred**; the foundational few are now baked into the rewrite design:
- **Baked in (binding in the ADRs, code pending):** IRI-template/term-construction **lifting** (join on raw key columns, build terms only in the final projection) + the **plan cache** (⟨T,M⟩-epoch invalidation, schema-selecting-constant keying) → **ADR-0007**; term-gen **allocation discipline** (`NamedNodeRef` zero-copy, `generate_into(&mut String)`, `new_unchecked`, precompiled template segments) + cross-source semi-join **cost** (side-selection + reducer sizing + skip-if-unselective) → **ADR-0006**.
- **Near-free:** the FTS baseline `FILTER(CONTAINS/STRSTARTS/REGEX) → ILIKE/~*`.
- **Reserve seams (don't build):** an extension-function/predicate registry, a `geo:wktLiteral` datatype hook, an FTS-index registry — so FTS / GeoSPARQL bolt on later without rework.
- **Defer until gated:** FTS property-function, GeoSPARQL push-down, result-cache (out-of-engine via HTTP validators), JUCQ-cover factoring, SIMD proper, T-mapping disk cache, fast allocator (measure first).
- **Out of scope:** external `SERVICE` federation.
